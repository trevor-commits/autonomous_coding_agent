# Контекст продукта

## Аудитория

Trevor as the operator running and tightening the harness, plus later audit
sessions that need a durable repo-visible explanation of what the system is and
what it is not.

## Боль

Autonomous coding systems are easy to oversell and hard to trust. Without
deterministic legality, bounded scope, and artifact-backed closeout, the system
turns into prompt theater instead of a usable operator tool.

## Польза

A narrow harness that can execute bounded coding work honestly, verify it
locally, and explain the outcome with real artifacts.

## Пользовательские сценарии

- As an operator, I want to choose a bounded coding task or queue issue and get
  a truthful ready/blocked outcome with evidence instead of vague agent claims.
- As an auditor, I want the repo and Linear trail to show what was planned,
  what was executed, and what remains blocked.

## Что не делаем сейчас

- broad hosted control plane
- multi-writer concurrency
- auto-merge/auto-deploy
- Phase 4 and Phase 5 expansion before one real repo is proven
