# Продуктовая аналитика

## Аудитория

The audience is operator-first. They care about control, evidence, narrow
scope, honest blockers, and clear ownership more than they care about flashy AI
positioning.

## Главный сценарий

Choose a real repo task -> let the deterministic supervisor normalize and bound
it -> let Codex implement inside that boundary -> run deterministic and UI
verification -> inspect a ready/blocked artifact trail.

## Главная ценность

It turns autonomy from "maybe the model did something useful" into a bounded,
reviewable, reproducible delivery loop.

## Главный CTA

Approve one bounded plan and, when appropriate, name the first real target repo
to prove the harness against.

## Что критично в первой версии

- one writer
- deterministic legality
- repo-visible closeout
- target-repo truth instead of generic platform scope

## Что можно отложить

- webhook-first wakeups after the first proof
- bounded Claude strategy
- memory/resume hardening
